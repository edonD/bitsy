**However, there are two bugs:**

**Bug 1: Tiered strategy applies batch discount to Perplexity incorrectly.**

In the non-tiered path (line 219), batch discount is correctly gated per-model:
```typescript
if (useBatchAPI && model.batchDiscount < 1) {
  queryCost *= model.batchDiscount;
}
```

But in the tiered strategy path (line 260-264), a blanket `batchMult = useBatchAPI ? 0.5 : 1` is applied to ALL tier costs uniformly:
```typescript
const batchMult = useBatchAPI ? 0.5 : 1;
tieredMonthlyCost =
  totalQueries * budgetShare * cheapestBudget * batchMult +
  totalQueries * midShare * cheapestMid * batchMult +
  totalQueries * flagshipShare * cheapestFlagship * batchMult;
```

If a user selects only Perplexity Sonar as their mid-tier model (or only Perplexity Sonar Pro as flagship) and enables both Batch API + Tiered Strategy, the tiered calculation would apply a 50% discount to Perplexity's costs. But Perplexity has `batchDiscount: 1` (no batch discount). The non-tiered path handles this correctly; the tiered path does not.

**Bug 2: Tryscope preset label/value mismatch.**

```typescript
{ label: "Tryscope-Style (50 polls/day)", brands: 10, queries: 5, samples: 1, frequency: 1 },
```

The label says "50 polls/day" but `frequency` is set to `1`. Research 2.2 describes Tryscope as polling "50x/day." A user clicking this preset expecting Tryscope-level monitoring costs would see costs 50x lower than actual Tryscope usage. The `frequency` should be `50`, or the label should not claim "50 polls/day."

For a cost calculator, calculation correctness is the core contract. Bug 1 produces wrong numbers in a reachable configuration. This drops code quality to 1/2.
